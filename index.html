What's inside the box

Built for juggling, not context-switching.

  1. 01

    One window per Peti

    Open a project and you're inside it — two or three repos as draggable, resizable, translucent terminal cards over a full-bleed background. No switcher, no tab soup.

  2. 02

    Real Claude Code, your auth

    Every card is the genuine claude CLI in its repo. Insert / send prompt modes, bracketed-paste so multi-line never mis-submits, and --continue resume panes.

  3. 03

    Know who needs you

    Per-card working / awaiting badges read from Claude's own transcript — plus a menubar count across all Petis and a chime when one finishes.

  4. 04

    A plan that Claude can read

    A floating note per project holds a description and tasks with priority, labels, and a pinned Next up — mirrored one-way into .peti/PLAN.md in each pane's repo so Claude knows what's next.

  5. 05

    A dock for your cards

    A slim rail of every card — click to raise or reopen, and a ＋ to spawn a new Claude, shell, or read-only code-viewer card on the fly.

  6. 06

    Make it yours

    Per-Peti background (bundled wallpapers, gradient presets, or your own image, swapped live), an accent colour, and light / dark / system themes. Per-Peti launchers put a project one click from your Dock.

Build it from source.

You'll need Rust, Bun, and the claude CLI already logged in.

zsh — peti 
# clone & install
$ git clone https://github.com/ArnavPuri/peti.git
$ cd peti
$ bun install

# run in dev — or `bun run tauri build` to package
$ bun run tauri dev
